# 커밋 규칙

## 메시지 형식
[접두어] 한글 요약

- 본문(body) 없이 한 줄 요약만 작성
- 세부 설명이 필요하면 PR body에 기술

### 접두어
- `[feat]` — 새 기능
- `[fix]` — 버그 수정
- `[refactor]` — 리팩토링 (기능 변경 없음)
- `[doc]` — 문서 변경

## 브랜치 전략
- 메인 이슈: `feat/{메인이슈번호}-{작업내용}` (main에서 분기)
- 서브이슈: `feat/{메인이슈번호}-{순번}-{서브작업내용}` (메인이슈 브랜치에서 분기)
- 머지: 항상 `--no-ff` (merge commit 생성, 분기 기록 보존)
- pull: `rebase` (불필요한 머지 커밋 방지)

## PR 규칙
- 이슈:PR = 1:1
- 서브이슈 PR → 메인이슈 브랜치 (base)
- 메인이슈 PR → main (수동 머지)
- 머지 후 head 브랜치 자동 삭제
- body에 `Closes #{이슈번호}` 포함하여 이슈 자동 닫기

## 커밋 분할 원칙
- 하나의 커밋은 하나의 논리적 변경 단위 (라인 단위까지 분할)
- 시간순으로, 연관 있는 작업끼리 묶어서 커밋
- 예: 도메인 모델 변경 → 마이그레이션 → 설정 변경 → 문서 업데이트

## 금지 사항
- AI 도구 관련 문구 (Co-Authored-By, Generated with 등) 커밋/PR에 포함 금지
